Column width

In this example column-width have been set as 200px width.

The browser will fit as many as columns into the container as posible, using the specfied width as ideal width.

The specification says:

describes the optimal column width. The actual column width may be wider (to fill the available space), or narrower (only if the available space is smaller than the specified column width). Specified values must be greater than 0.

Column count

If you specify how many columns you want then the browser decide how wide the should be. In this example column-count is setted as 3.

The specification says:

describes the optimal number of columns into which the content of the element will be flowed. Values must be greater than 0. If both ‘column-width’ and ‘column-count’ have non-auto values, the integer value describes the maximum number of columns.

Column rule

This property allow you to style your columns with some styles (not very much and not very cool). The properties you can set are: column-rule-style column-rule-width column-rule-color.

This properties behave in the same way as border-style border-width border-color.

Column breaks

You can control how columns breaks with this property.

For example, if you want avoid breaks inside paragraphs and before blockquotes you can do this:


                    .col-wrapper p {
                        break-inside: avoid;
                    }
                    .col-wrapper bockquote {
                        break-before: avoid;
                    }
Remember use browser prefixes.
